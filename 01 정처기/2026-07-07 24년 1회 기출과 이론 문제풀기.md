### ❌ 틀린 문제 & 깨달은 점

1. **오버로딩 vs 오버라이딩**
   - 오버로딩(매개변수만 다르게) 컴파일(문법 검사)
   - 오버라이딩(상속 재정의) 런타임(프로그램 실행)

2. **네트워크 주소 구하기**
   - 네트워크 주소를 구한 다음에 호스트 비트 자리 같은 부분만 1로 채우면 브로드캐스트 주소가 됨
   - ex. 192.168.32.23/22 였다면 네트워크주소로 바꾼 다음 끝에서 10개까지를 1로 바꿔야 브로드캐스트 주소가나옴

3. **IPSec**
   - 네트워크 계층에서 IP 패킷 단위로 암호화와 인증을 수행하는 프로토콜
   - IP는 주소 지정및 경로 설정하지만 신뢰성은 보장하지 않는다.

4. **연산 구조 제약조건**
   - 데이터 모델링의 3요소

5. **외부 개념 내부 스키마**
   - 외부는 사용자가 볼거 개념은 논리적 내부는 물리적

6. **메일 까먹지 말기**
   - SMTP는 25번 메일
   - IMAP는 143번 전자메일 동기화

7. **웜은 네트워크 전파 바이러스는 네트워크 전파안함**
8. **스니핑은 엿듣기(패킷채가서몰래보기) 스푸핑은 속이기(주소속이기)**

9. **네트워크 7계층 물데네전세표응**
  - 물리: 비트, 케이블 리피터 허브, 눈에 보임
  - 데이터링크: 프레임, 이더넷 MAC 스위치 브리지 PPP, 인접한 장비끼리 데이터 주고받는 연결 관리
  - 네트워크: 패킷, IP ICMP ARP 라우터, 여러 네트워크 거쳐 목적지까지(네트워크 간 경로)
  - 전송: 세그먼트, TCP UDP, 데이터를 목적지까지 확실하게
  - 세션: 메시지, 세션 관리 동기화, 대화 시작할 때 연결 유지 단위
  - 표현: 메시지, 암호화 압축 인코딩, 이 데이터가 어떤 모습이야? 어떻게 보여줘?
  - 응용: 매시지, HTTP FTP SMTP DNS Telnet, 실제 사용하는 응용 프로그램과 맞닿음
  - 캡슐화: 계층 내려갈 때 각 계층 헤더 붙는 과정
  - **IP 주소 관여나 네트워크 간 이동/경로면 걍 3계층(네트워크)**

10. **라우팅 프로토콜**
  - RIP 거리벡터 15홉 소규모
  - OSPF 다익스트라 링크상태 대규모
  - BGP AS(자율시스템)간 라우팅 인터넷 백본

11. HDLC
  - 동기식 전송 방식
  - 비트 스터핑으로 오류제어
  - ISU 이수! 정보(데이터전송) 감독(흐름오류제어) 비번호(링크설정해제) 프레임
  - NRM 주국-종국 ARM 종국 허가없어도 가능 ABM 모든 국 대등
  - PPP는 인증 기능 지원(PAP, CHAP), 에러체크, 두 지점 간 직접 연결은 PPP
