### ❌ 틀린 문제 & 깨달은 점

> 문제 통째로 적지 말고, '내가 몰랐던 개념/핵심'만 한 줄로 요약

1. 지연 갱신 VS 즉각 갱신
   - **헷갈린 점:** 용어가 헷갈림 기억이 안 남
   - **한줄 정리:** 지연 갱신은 완료될 때 까지 기다림, 즉각 갱신은 바로 개입함
   - **연결 개념:** 로그 기반 회복 - 데이터베이스 변경 사항 로그 파일에 순차 기록,
			장애 발생 시 복구

2. 그림자 페이징
   - **헷갈린 점:** 수정 시 원본(그림자) 페이지 안 건드리고 새 페이지에 기록한다. 현재 페이지 테이블만 새 페이지를 가리킨다.
   - **한줄 정리:** 그림자페이지는 원본을 꼬옥 갖고있어주면 돼~ 든든한 백업
   - **연결 개념:** 페이지 - 데이터 묶음
			페이지 테이블 - 주소록(몇 번 페이지 디스크 어디에 저장되있는지 적어둔 표)

3. SQL문
   - **헷갈린 점:** 그냥 모든게 다 헷갈림 처음부터 다시 읽으면서 문제 풀어야 함
   - **한줄 정리:** DDL 정의어 ALTER 테이블 DML 조작어 SELECT 데이터 DCL 제어어 GRANT 권한
   - **연결 개념:** CONSTRAINT FOREIGN KEY REFERENCES

 4. 데이터 정의어 조작어 제어어 헷갈림
   - **헷갈린 점:** DELETE나 DROP이나 ALTER나... 뭐랄까 뭐가 뭔지 모르겠달까
   - **한줄 정리:** DDL 정의어 ALTER 테이블 DML 조작어 SELECT 데이터 DCL 제어어 GRANT 권한
   - **연결 개념:** CASCADE / RESTRICT -> 연쇄 회수/회수 거부(옵션 생략 시 기본)

### ❌ 틀린 문제 & 깨달은 점

1. 디비젼(**A÷B**)
   - **헷갈린 점:** B 릴레이션의 **모든 값**과 짝을 이루는 A의 값만 골라내라.
   - **한줄 정리:** **모든 값**과 짝을 이뤄야 함. 그 중 몇 개만 만족 X
   - **연결 개념:** #관계대수연산자

2. 데이터 모델 3가지 구성요소
   - **헷갈린 점:** 구조, 연산, 제약조건 개념 자체
   - **한줄 정리:** 구조는 어떻게 나타낼 것인지 표현, 연산은 데이터 처리 작업 명세, 제약조건은 데이터 무결성 유지를 위한 제약
   - **연결 개념:** #데이터모델

3. 교집합∩과 조인**⋈**
   - **헷갈린 점:** 개념 차이
   - **한줄 정리:** 교집합은 위아래로 합침(공통된 행 찾아내기), 조인은 옆으로 붙임(새로운 정보 만들기)
   - **연결 개념:** 합집합 u, 차집합 -, 카티션 프로덕트 x (행렬) #관계대수연산자 

4. 정규형 
   - **헷갈린 점:** 개념
   - **한줄 정리:** 1) 하나셀 단일값만 2) 기본키 2개 이상일 때 3) 기본키 아닌 놈들끼리 꼬리물기 분리해서 제거 4) BCNF 결정자(식별자)인데 후보키(유일, 최소)가 아니라면 쪼개기 
   - **연결 개념:** 유일성(중복 금지), 최소성(군더더기 금지) #정규화 

5. 해싱 접근
  - **헷갈린 점:** 해싱 접근, 해시 충돌, 해시 주소, 해시 함수 -> 용어
  - **한줄 정리:** "해싱 접근"외 다 해시
  - **연결 개념:** 레드 블랙 트리 체이닝(Java) #레드블랙트리

6. 관계 대수 vs 관계 해석
  - **헷갈린 점:** 관점따라 도메인의 뜻이 다름, 관계 대수 - 어떻게 계산할까? How 관계 해석 - 어떤 데이터? What
  - **한줄 정리:**
	실무-이름표: 이 데이터는 무슨 정보인가?
	수학-울타리: 이 칸에 어떤 값만 들어올 수 있나?
	관계 해석-빈칸(x,y): 도메인 변수 조건식 짤 때 이 칸의 값을 뭐라고 부를까?
  - **연결 개념:** 절차적 언어(관계 대수), 비절차적 언어(관계 해석, SQL)
